## 만 개의 DOM 노드, 당신의 리액트 앱을 멈추게 하는가? 리스트 가상화의 모든 것!

### 소개

사용자 경험(UX)은 프론트엔드 개발의 핵심이죠. 특히 대량의 데이터를 리스트 형태로 보여줘야 할 때, 성능 문제는 개발자를 항상 괴롭히는 단골손님입니다. 화면에 보이지 않는 수천, 수만 개의 DOM 노드가 앱의 속도를 저하시키는 현상, 경험해보셨나요? 오늘은 이 고질적인 문제를 해결할 리스트 가상화(List Virtualization)에 대해 깊이 파고들어 보겠습니다.

### 본문

프론트엔드 개발에서 대량의 데이터를 효율적으로 렌더링하는 것은 핵심적인 성능 최적화 과제입니다. 단순히 데이터가 많다고 해서 느려지는 것이 아니라, *DOM 노드의 수가 많아질수록* 브라우저의 렌더링 엔진에 부하가 가해져 앱 전반의 반응성이 떨어지게 됩니다. 심지어 화면에 보이지 않는 DOM 노드까지도 브라우저 메모리와 CPU에 부담을 주어 성능에 영향을 미치죠.

이 문제를 해결하기 위한 대표적인 접근 방식은 다음과 같습니다.

1.  **페이지네이션 (Pagination):**
    전통적인 방식입니다. 데이터를 일정 단위로 나누어 페이지별로 보여줍니다. 사용자는 '다음' 버튼을 눌러 다음 페이지를 로드합니다.

    *   **장점:** 구현이 비교적 간단하고, 서버 부하를 줄일 수 있으며, 사용자가 특정 페이지로 쉽게 이동할 수 있습니다. 데이터의 전체 규모를 파악하기 용이합니다.
    *   **단점:** 사용자 흐름이 끊기고, 페이지 전환 시마다 새로운 요청과 렌더링이 발생하여 매끄럽지 못한 경험을 줄 수 있습니다.

2.  **무한 스크롤 (Infinite Scroll):**
    사용자가 스크롤을 내리면 추가 데이터를 로드하여 기존 리스트에 이어 붙이는 방식입니다. 페이스북, 인스타그램 등 많은 서비스에서 사용됩니다.

    *   **장점:** 사용자 흐름이 자연스럽고, 페이지 전환 없이 연속적인 탐색이 가능합니다. 초기 로드 시 최소한의 데이터만 가져와 빠르게 화면을 보여줄 수 있습니다.
    *   **단점:** 모든 데이터를 결국 DOM으로 렌더링하므로, 리스트가 길어질수록 DOM 노드 수가 기하급수적으로 늘어나 성능 저하를 피할 수 없습니다. 특히 긴 리스트에서 위로 스크롤하여 특정 항목을 찾기 어렵거나, '맨 위로' 버튼이 없으면 끝없이 스크롤해야 하는 불편함이 있습니다.

3.  **리스트 가상화 (List Virtualization) / 가상 스크롤 (Virtual Scroll):**
    오늘의 주인공입니다. 화면에 *실제로 보이는* 아이템과 약간의 버퍼 아이템만 DOM으로 렌더링하고, 나머지는 렌더링하지 않는 방식입니다. 사용자가 스크롤하면 보이는 영역에 따라 동적으로 DOM 노드를 생성하고 파괴합니다. 마치 커다란 리스트의 작은 '창문'을 통해 데이터를 보는 것과 같다고 하여 "Windowing"이라고도 불립니다.

    *   **장점:** 수만, 수십만 개의 데이터도 부드럽게 렌더링할 수 있습니다. DOM 노드 수가 항상 일정하게 (화면에 보이는 개수 + 버퍼) 유지되어 압도적인 성능을 제공합니다. 메모리 사용량과 CPU 부하를 현저히 줄일 수 있습니다.
    *   **단점:** 구현이 페이지네이션이나 무한 스크롤보다 복잡하며, 아이템 높이가 가변적인 경우 처리하기 까다로울 수 있습니다. 각 아이템의 크기를 사전에 알거나 추정할 수 있어야 효율적입니다.

#### 실제 코드 예시 (React + TypeScript)

리액트 환경에서 리스트 가상화를 구현하는 가장 일반적인 방법은 `react-window` 또는 `react-virtualized`와 같은 라이브러리를 활용하는 것입니다. 여기서는 더 가볍고 사용하기 쉬운 `react-window`를 사용한 예시를 살펴보겠습니다.

먼저, `react-window`를 설치합니다:

```bash
npm install react-window

yarn add react-window
```

다음은 `FixedSizeList` 컴포넌트를 사용하여 10,000개의 아이템을 가상화하여 렌더링하는 예시입니다.

```tsx
// src/components/VirtualListExample.tsx
import React from 'react';
import { FixedSizeList } from 'react-window';

// 더미 데이터 생성
interface Item {
  id: number;
  text: string;
  description: string;
}

const generateItems = (count: number): Item[] => {
  return Array.from({ length: count }, (_, i) => ({
    id: i,
    text: `아이템 번호 ${i + 1}`,
    description: `이것은 ${i + 1}번째 아이템에 대한 설명입니다. 대량의 데이터를 효율적으로 처리하는 가상화 리스트의 위력을 느껴보세요.`,
  }));
};

const items = generateItems(10000); // 1만 개의 아이템

// 리스트의 각 행을 렌더링할 컴포넌트
interface RowProps {
  index: number;
  style: React.CSSProperties; // react-window가 자동으로 전달하는 스타일 객체
}

const Row: React.FC<RowProps> = ({ index, style }) => {
  const item = items[index];
  return (
    <div
      style={{
        ...style,
        display: 'flex',
        flexDirection: 'column',
        justifyContent: 'center',
        borderBottom: '1px solid #eee',
        padding: '10px 15px',
        backgroundColor: index % 2 ? '#f9f9f9' : '#ffffff',
      }}
    >
      <h3 style={{ margin: 0, fontSize: '1.1em' }}>{item.text}</h3>
      <p style={{ margin: '5px 0 0 0', fontSize: '0.9em', color: '#666' }}>{item.description}</p>
    </div>
  );
};

const VirtualListExample: React.FC = () => {
  return (
    <div style={{ maxWidth: '600px', margin: '20px auto', border: '1px solid #ddd', borderRadius: '8px', overflow: 'hidden' }}>
      <h2 style={{ padding: '15px', margin: 0, backgroundColor: '#f0f0f0', borderBottom: '1px solid #ddd' }}>
        10,000개 아이템을 가상화하여 렌더링
      </h2>
      <FixedSizeList
        height={400} // 리스트의 전체 높이
        width="100%" // 리스트의 전체 너비
        itemCount={items.length} // 총 아이템 수
        itemSize={70} // 각 아이템의 고정 높이 (픽셀)
      >
        {Row}
      </FixedSizeList>
    </div>
  );
};

export default VirtualListExample;
```

위 코드에서 `FixedSizeList`는 `height`, `width`, `itemCount`, `itemSize`를 필수로 받습니다. `itemSize`는 각 아이템의 고정된 높이를 지정하며, 이를 통해 `react-window`는 렌더링할 아이템의 위치와 크기를 효율적으로 계산합니다. `Row` 컴포넌트는 `index`와 `style` prop을 받아서 해당 인덱스의 아이템을 렌더링하고, `style` 객체를 적용하여 올바른 위치에 배치됩니다. 이 `style` 객체에는 `position: absolute`, `top`, `height`, `width` 등의 속성이 포함되어 가상화가 동작하도록 합니다.

#### 실무 적용 팁

1.  **언제 가상화를 선택해야 할까?**
    *   **페이지네이션:** 데이터의 무결성이 중요하고, 사용자가 특정 페이지로 이동하는 것이 자연스러운 경우 (예: 관리자 페이지의 주문 목록, 검색 결과).
    *   **무한 스크롤:** 탐색 위주의 경험이 중요하고, 사용자가 끝까지 스크롤할 가능성이 낮은 경우 (예: 뉴스 피드, 블로그 목록). 단, 데이터 양이 너무 많아지면 결국 성능 병목이 발생할 수 있음을 인지해야 합니다.
    *   **가상화:** 수백 개 이상의 아이템이 예상되며, 아이템들의 높이가 비교적 균일하거나 예측 가능한 경우 (예: 채팅 목록, 파일 탐색기, 대규모 테이블). 압도적인 성능이 최우선일 때 선택합니다.

2.  **가변 높이 아이템 처리:**
    `react-window`의 `FixedSizeList`는 아이템 높이가 고정될 때 가장 효율적입니다. 만약 아이템 높이가 가변적이라면 `VariableSizeList`를 사용할 수 있습니다. 이 경우, 각 아이템의 높이를 반환하는 `getItemSize` 함수를 제공해야 합니다. 정확한 높이를 알기 어렵다면, 캐싱(memoization) 기법을 사용하거나 높이를 추정하여 점진적으로 업데이트하는 전략을 고려해야 합니다. `react-virtualized`는 이 부분에서 더 유연한 옵션을 제공하기도 합니다.

3.  **접근성 (Accessibility):**
    가상화는 DOM 노드를 동적으로 추가/제거하므로 스크린 리더와 같은 보조 기술에 문제가 될 수 있습니다. `aria-live` 영역을 사용하거나, 리스트의 전체 크기를 가상으로 알리는 `aria-rowcount`, `aria-colcount` 등의 속성을 적절히 활용하여 접근성을 보장해야 합니다. `react-window`와 같은 라이브러리는 기본적인 접근성 고려 사항을 내장하고 있지만, 추가적인 커스터마이징이 필요할 수 있습니다.

4.  **로딩 스켈레톤:**
    데이터 로딩 중에는 빈 공간이 보이거나 아이템이 갑자기 나타나는 현상이 발생할 수 있습니다. 사용자 경험을 위해 로딩 중임을 시각적으로 보여주는 스켈레톤 UI를 적용하는 것이 좋습니다. 가상화된 리스트에서도 로딩 상태를 명확히 표시하여 사용자가 기다림을 인지할 수 있도록 해야 합니다.

5.  **성능 프로파일링:**
    개발자 도구의 성능 탭을 활용하여 렌더링 시간, 레이아웃 계산, DOM 노드 수 등을 주기적으로 확인하고 최적화 지점을 찾아야 합니다. 가상화 적용 전후의 성능 지표를 비교하여 개선 효과를 명확히 파악하는 것이 중요합니다.

### 마무리

오늘 우리는 대규모 리스트 렌더링의 고질적인 성능 문제를 해결하기 위한 세 가지 전략, 특히 리스트 가상화의 강력함에 대해 알아보았습니다. 만 개의 DOM 노드가 더 이상 여러분의 리액트 앱 성능을 저하시키는 주범이 되지 않도록, 적절한 상황에 맞는 최적화 기법을 적용하는 것이 중요합니다. 특히 `react-window`와 같은 라이브러리를 활용하면 적은 노력으로도 놀라운 성능 개선을 이룰 수 있습니다.

다음 단계로는 `VariableSizeList`를 사용하여 가변 높이 아이템을 처리하는 방법이나, 더 복잡한 그리드 형태의 가상화를 구현하는 `react-virtualized` 라이브러리를 탐구해보시는 것을 추천합니다. 사용자에게 끊김 없는 부드러운 경험을 제공하기 위한 여정은 계속됩니다! Happy Coding!
